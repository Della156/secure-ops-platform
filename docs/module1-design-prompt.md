# 模块一：网络安全自动任务配置 — 设计指导 Prompt

## 概述

本文档用于指导 AI 代码生成工具（trae）完成「网络安全自动任务配置」模块的全部前端页面设计与实现。该模块是整个系统的核心基础模块，覆盖自动化任务从注册、存储、编排、调度、执行到资源对接的全生命周期管理。

> ⚠️ 菜单配置和页面路由已在 `menuData.ts` 和 `pageRegistry.tsx` 中完成。trae **不需要**修改这两个文件，只需按下方指定路径创建/完善各个页面组件。

## 设计规范

所有页面必须遵循以下统一规范：

### 1. 技术栈
- **框架**: Next.js 14 (App Router, 'use client')
- **语言**: TypeScript (严格模式)
- **样式**: Tailwind CSS (暗色主题)
- **图标**: lucide-react
- **图表**: recharts（统计类页面）
- **组件引用**: `@/components/ui/Modal`, `@/components/ui/Toast`

### 2. 全局设计令牌 (Design Tokens)

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 页面背景 | `#0B1120` | 最底层背景 |
| 卡片/容器背景 | `#1A2332` | 主要内容区背景 |
| 卡片内嵌背景 | `#20293F` | 卡片内的次级区域 |
| 边框 | `#2A354D` | 分割线和边框 |
| 主文字 | `#F3F4F6` | 标题/正文 |
| 次文字 | `#9CA3AF` | 辅助说明 |
| 强调 | `#6366F1` | 按钮/链接/激活态 |
| 成功 | `#10B981` | 成功状态 |
| 警告 | `#F59E0B` | 警告状态 |
| 危险 | `#EF4444` | 错误/失败 |

### 3. 页面通用布局

每个页面遵循以下布局模式：

```
┌─────────────────────────────────────────────┐
│  页面标题行                                    │
│  ┌─────────────────────────────────────────┐ │
│  │  搜索/筛选栏（如需要）                     │ │
│  └─────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────┐ │
│  │  统计卡片行（如需要，3-5个关键指标卡片）     │ │
│  └─────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────┐ │
│  │  操作按钮 + 表格                         │ │
│  │  (分页、多选、批量操作)                    │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- **标题行**: 页面标题（text-xl font-bold）在左上角，右上角可放置主操作按钮
- **筛选栏**: 灰色背景卡片，内部 flex-wrap 布局，Input + Select + DatePicker + 搜索/重置按钮
- **统计卡片**: 3-5个指标卡片在一行（grid grid-cols-1 md:grid-cols-3 lg:grid-cols-5 gap-4）
- **表格**: overflow-x-auto 包裹，白色表头加灰色背景，行交替色
- **分页**: 表格底部，显示总数 + 页码切换

### 4. CRUD 操作规范

- **新增/编辑**: 使用 `Modal` 组件（`@/components/ui/Modal`），表单字段受控（`value` + `onChange`）
- **删除**: 二次确认使用自定义 Confirm 对话框（不要用 `confirm()`），成功/失败使用 `Toast`
- **表单**: 使用 `value` 而非 `defaultValue`，每个字段有 label 和校验提示
- **搜索**: 支持模糊搜索 + 状态筛选 + 时间范围
- **导出**: 表格数据导出为 CSV/Excel 功能

### 5. 状态管理

- 使用 `useState` 管理当前页面状态（列表数据、筛选条件、分页、弹窗可见性）
- 使用 `useEffect` 初始化数据加载（模拟API调用，填充示例数据）
- 使用 `useCallback` 优化回调函数

## 各子模块详细设计

### 1. 自动化任务注册与接入（taskRegistration/）

**菜单ID**: menu-1-1-1 ~ menu-1-1-3

#### 1.1 任务接入管理（TaskAccessManagement.tsx）
- **功能**: 任务接入配置的新增、修改、删除、查询；接入协议配置；接入参数配置
- **页面**: 表格展示已接入任务列表，每行显示任务名称、接入协议类型（SSH/API/Snmp/其他）、接入状态（已接入/未接入/异常）、最后检测时间
- **表格列**: 任务名称 | 接入协议 | 接入地址 | 接入状态 | 最后检测时间 | 操作（编辑/删除/测试连接）
- **弹窗**: 接入配置表单（任务名称、协议类型、连接地址、端口、认证凭据、超时设置、重试次数）

#### 1.2 任务在线注册管理（TaskOnlineRegistration.tsx）
- **功能**: 任务注册申请、注册信息校验、注册审批流程配置、注册状态管理（已注册/已注销）
- **页面**: 列表展示 + 注册申请弹窗
- **表格列**: 任务ID | 任务名称 | 申请部门 | 申请人 | 注册时间 | 状态（待审批/已注册/已注销） | 操作
- **弹窗**: 注册申请表单（任务名称、描述、所属分类、接口信息、审批人选择）

#### 1.3 任务接入状态管理（TaskAccessStatus.tsx）
- **功能**: 接入状态实时展示、状态历史查询、状态变更记录、接入失败告警提示、手动刷新
- **页面**: 卡片式状态概览 + 状态历史表格
- **指标卡片**: 总接入数、在线数、离线数、异常数
- **表格列**: 任务名称 | 接入状态 | 最近在线时间 | 状态变更时间 | 操作（刷新/查看详情）

## 示例数据规范

所有页面在开发阶段使用**模拟数据**（mock data）。每条模拟数据应包含合理的字段值，不要空置。

### 示例数据模式（所有页面通用）

```typescript
interface PageState<T> {
  data: T[];
  loading: boolean;
  searchParams: { keyword: string; status?: string; startDate?: string; endDate?: string };
  pagination: { current: number; pageSize: number; total: number };
  modalVisible: boolean;
  editingRecord: T | null;
}
```

### 命名约定
- 组件文件名: `PascalCase.tsx`
- 导出的组件名: `export function PascalCase()`
- 类型/接口: `interface XxxItem` 或 `type XxxItem`
- 状态变量: `xxxList, setXxxList` / `loading, setLoading`

### Mock 数据生成
- 每个页面至少有 20 条预置数据
- ID 使用 `uuid` 格式或自增数字字符串
- 时间字段使用 `YYYY-MM-DD HH:mm:ss` 格式
- 状态字段用枚举字符串，对应颜色的状态标签组件
