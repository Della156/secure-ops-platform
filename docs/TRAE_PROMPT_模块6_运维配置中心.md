# Trae AI 设计提示词：模块6 — 运维配置中心

## 项目概况

这是一个**网络安全智能化运维平台**的前端项目，使用 Next.js + TailwindCSS + TypeScript 构建。
当前需要你开发 **模块6：运维配置中心** 下的所有三级页面。该模块的目标是：
> 建设平台自身的省级属地化配置与运维支撑功能，具备数据同步配置、运行参数配置、数据字典配置、数据供给配置与监控、访问行为配置及备份恢复配置等运维配置能力。

## 你的任务

根据本文档中的功能描述，在已有项目中创建 **模块6** 的以下页面组件。
该模块共有 **13 个二级菜单**，每个对应一个目录和若干三级页面。
你需要：
1. 在 `src/pages/module6/` 目录下创建所有三级页面组件
2. 在 `src/data/menuData.ts` 的 `menu-6` 下补充三级菜单配置
3. 每个页面使用 'use client'，遵循深色主题设计规范

## 系统已有文件结构
```
src/
  data/menuData.ts              ← 菜单数据（含三级菜单结构）
  types/index.ts                ← MenuItem / SubMenuItem / ThirdMenuItem 类型
  components/
    Layout/MainLayout.tsx       ← 主布局
    Sidebar/Sidebar.tsx         ← 侧边栏容器
    Sidebar/MenuItem.tsx        ← 菜单项组件（支持三级展开）
    Pages/PageContent.tsx       ← 页面内容占位区域

    ui/Toast.tsx                ← 提示组件
    index.ts
  contexts/SystemContext.tsx    ← 系统上下文（含 activeMenu 状态）
  app/
    layout.tsx                  ← 根布局（含 SystemProvider）
    page.tsx                    ← 首页（PageContent）

```

## 设计规范

### 1. 色彩体系
```css
--primary: #3b82f6    /* blue-500 */
--primary-light: #60a5fa  /* blue-400 */
--primary-dark: #2563eb   /* blue-600 */
--bg-card: #1e293b     /* slate-800 */
--bg-surface: #0f172a  /* slate-900 */
--bg-page: #020617     /* slate-950 */
--text-primary: #f8fafc   /* slate-50 */
--text-secondary: #94a3b8 /* slate-400 */
--text-muted: #64748b   /* slate-500 */
--border: #334155     /* slate-700/50 */
--success: #22c55e    /* green-500 */
--warning: #eab308    /* yellow-500 */
--danger: #ef4444     /* red-500 */
--info: #06b6d4       /* cyan-500 */
```

### 2. 组件风格
- 卡片容器：`bg-slate-900 border border-slate-800 rounded-xl`
- 输入框：`bg-slate-800 border border-slate-700 rounded-lg text-white`
- 按钮（主）：`bg-blue-600 hover:bg-blue-700 text-white rounded-lg px-4 py-2`
- 按钮（次）：`bg-slate-800 hover:bg-slate-700 text-slate-300 rounded-lg px-4 py-2`
- 表格：`w-full border-collapse` 表头 `bg-slate-800/50 text-slate-400` 行 `hover:bg-slate-800/30`
- 标签/徽章：`bg-blue-500/20 text-blue-400`、`bg-green-500/20 text-green-400` 等
- 搜索框：左侧搜索图标 + 输入框，`bg-slate-800 border border-slate-700`
- 分页：底部居中，页码按钮 `bg-slate-800 hover:bg-slate-700`

### 3. 页面通用布局
每页布局结构（从上到下）:
```
┌─ 页面标题（二级/三级菜单名称）
├─ 操作栏（搜索框 + 新增按钮 + 批量操作等）
├─ 筛选区（下拉选择器、日期范围等）
├─ 数据表格（列表展示）
├─ 分页（底部）
└─ 弹窗（新增/编辑/详情 Modal，使用 Modal 组件）
```

### 4. 状态管理
- 使用 `useSystem().activeMenu` 获取当前激活菜单
- 菜单路由 ID 格式：`menu-6-x-y`（一级-二级-三级）
- 当前页面用 `{children}` 渲染在 `MainLayout` 中

### 5. 通用功能模式
所有 CRUD 页面通用的操作模式：
- **新增**：右上角「+ 新增」按钮 → 弹出 Modal 表单 → 填写 → 提交 → 刷新列表
- **编辑**：表格行操作列「编辑」按钮 → 弹出 Modal 表单（带初始值）→ 修改 → 提交
- **删除**：表格行操作列「删除」按钮 → Confirm 确认 → 删除 → 刷新列表
- **查询**：顶部搜索框 + 筛选条件 → 点击「查询」→ 更新列表
- **导出**：列表顶部「导出」按钮 → 导出为 Excel/CSV
- **详情查看**：点击行或「详情」按钮 → 弹出详情 Modal 或跳转详情页

## 技术要求

1. 使用 **Next.js App Router** 模式（`src/app/` 目录）
2. 全部使用 **'use client'** 组件
3. 使用 **TailwindCSS** 样式（使用预设的深色主题色板）
4. 图标使用 **lucide-react**
5. 表格、表单、弹窗等通用组件如果已有则复用，否则自行实现
6. 图表推荐使用 **recharts** 库
7. 所有代码保持中文注释
8. 遵循现有项目的文件命名和代码风格约定
